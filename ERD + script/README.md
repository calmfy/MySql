# 📚 pet_db — База данных "Питомцы"

## 📌 Описание проекта

В рамках практической работы была разработана реляционная база данных для компании, предоставляющей услуги по уходу за питомцами (выгул собак, стрижка кошек).

База данных позволяет:

* хранить информацию о владельцах животных
* учитывать питомцев (кошки, собаки)
* хранить данные об исполнителях
* вести учет заказов
* формировать отчёты
* учитывать прививки животных (доработка)

## 🚀 Инструкция по запуску

### 1. Создание базы данных

```sql
CREATE DATABASE pet_db;
USE pet_db;
```

---

### 2. Запуск основного скрипта

Открыть файл:

```
Pets_script_mysql.sql
```

Выполнить его в MySQL Workbench.

📌 Результат:

* создаются основные таблицы:

  * owner
  * pet
  * employee
  * order1
  * service
  * и др.

---

### 3. Проверка базы данных

```sql
SELECT * FROM pet;
SELECT * FROM owner;
SELECT * FROM order1;
```

---

## 🔧 Доработка: учёт прививок

Добавлены новые сущности:

* `vaccine_type` — вид прививки
* `vaccine` — информация о прививках

---

### 4. Запуск скрипта прививок

Открыть файл:

```
vaccine_patch_mysql.sql
```

Выполнить его.

📌 Результат:

* создаются таблицы:

  * vaccine
  * vaccine_type
* добавляются тестовые данные

---

### 5. Проверка

```sql
SELECT * FROM vaccine;
SELECT * FROM vaccine_type;
```

---

## 🔄 Удаление доработки (перед сдачей)

Перед демонстрацией преподавателю необходимо удалить таблицы прививок:

```sql
USE pet_db;

DROP TABLE vaccine;
DROP TABLE vaccine_type;
```

После этого можно повторно запустить скрипт `vaccine_patch_mysql.sql` для демонстрации.

## 📊 ER-модель

База данных включает следующие сущности:

* Owner (владелец)
* Pet (питомец)
* Employee (исполнитель)
* Order (заказ)
* Service (услуга)
* Vaccine (прививка)
* VaccineType (вид прививки)

Связи:

* Owner 1 → N Pet
* Pet 1 → N Order
* Employee 1 → N Order
* Pet 1 → N Vaccine
* VaccineType 1 → N Vaccine
